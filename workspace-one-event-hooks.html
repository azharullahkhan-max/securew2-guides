📡 Introduction

How Event Hooks connect Workspace ONE to SecureW2

Workspace ONE's Event Notifications feature sends outbound HTTP calls to SecureW2 whenever a defined device lifecycle event occurs. SecureW2 receives these notifications and takes automated action — such as revoking the certificate associated with the affected device.

ℹ️
Why this matters Without event-driven revocation, a certificate issued to a wiped or deleted device remains valid until its natural expiry. Event Hooks close that gap — the moment Workspace ONE processes a device wipe or deletion, SecureW2 revokes the certificate and the device loses 802.1X network access immediately.

✅ Prerequisites

What you need before you begin

Required

  • Active Workspace ONE portal subscription with Admin rights
  • SecureW2 JoinNow Management Portal account with Cloud Connector and JoinNow Real-Time Intelligence subscriptions active

⚙️ Configuring Event Hooks in JoinNow

Generate the endpoint URL and authentication secret

The JoinNow side of configuration produces two things you will need for Workspace ONE: a Notification URL and a Secret. Complete these steps first.

1
Log in to the JoinNow Management Portal

Use your admin credentials at your organisation's JoinNow portal URL.

2
Navigate to Device Control Platforms

Go to Integrations Hub → Device Control Platforms.

3
Add a new integration

Click Add. Enter a Name and Description for this Workspace ONE event hook, then click Save. The page will refresh and open a Configurations tab.

4
Copy the Notification URL

On the Configurations tab, copy the Notification URL and paste it into a text editor. You will use this when configuring the Workspace ONE endpoint.

5
Download the Authentication Secret

Click Download to save the .csv file containing the Authentication Secret. Keep this file secure — it authenticates Workspace ONE's calls to SecureW2.

6
Configure revocation events

Under the Events section, select Revoke the associated certificate for the events you want to act on:

  • Device Delete — revoke when a device record is removed from Workspace ONE
  • Device Wipe — revoke when a remote wipe command is issued

Select one or both as appropriate for your policy.

7
Save the configuration

Click Update to save your event hook settings in JoinNow.

✅
JoinNow setup complete. You now have a Notification URL and Authentication Secret ready for use in Workspace ONE.

☁️ Configuring Event Notifications in Workspace ONE

Create the event notification rule that calls SecureW2

⚠️
Have your Notification URL and Secret ready Complete the JoinNow configuration steps above before proceeding. You will need both values during this section.
1
Log in to the Workspace ONE portal

Use an account with Admin rights.

2
Open All Settings

Navigate to Groups & Settings → All Settings.

3
Navigate to Event Notifications

Under System, click Advanced, then go to API → Event Notifications.

4
Add a new rule

Click ADD RULE. The Add Event Notification dialog will open.

5
Set the Target Name

In the Target Name field enter a descriptive name, e.g. SecureW2 Certificate Revocation.

6
Build the Target URL

Open the .csv file downloaded from JoinNow and copy the Authentication Secret. Construct the Target URL by inserting the secret into the Notification URL at the placeholder position:

https://api.securew2.com/urlauth/<Authentication Secret>/V1/notify

Paste the complete URL into the Target URL field in Workspace ONE.

7
Select the trigger events

Scroll down to the event list and enable the following recommended options:

  • Device Delete
  • Device Unenrolled Enterprise Wipe
  • Device Wipe
8
Save the rule

Click Save. The event notification rule is now active in Workspace ONE.

✅
Setup complete. When a device is deleted or wiped in Workspace ONE, SecureW2 will receive the notification and automatically revoke the associated certificate. The device will lose access to the secure Wi-Fi network.
